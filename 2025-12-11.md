### 클래스가 필요한 이유
1. 배열을 사용해서 관리하게 되었을 때 만약 배열의 여러개가 서로 연결되어 있다면 그 중 하나의 데이터를 관리하게 되었을 때 데이터 불일치가 생기지 않도록 여러 배열들의 인덱스 순서를 항상 정확하게 맞춰줘야 한다.
  컴퓨터가 코드를 볼 때는 아무 문제 없지만 사람이 관리하기에는 어려움이 있다. 
  그렇기 때문에 하나의 개념으로 같이 묶는 것이 좋다.
2. 클래스에 정희한 변수들을 멤버 변수, 또는 필드라고 한다.
  멤버 변수(Member Variable): 이 변수들은 특정 클래스에 소속된 멤버이기 때문에 이렇게 부른다.
  필드(Field): 데이터 항목을 가르키는 전통적인 용어이다. 데이터베이스, 엑셀 등에서 각각의 항목을 필드라 한다.
  클래스에 소속된 변수를 뜻한다.
3. 클래스는 관례상 대문자로 시작하고 낙타 표기법을 사용한다.

### 클래스와 사용자 정의 타입
- 타입은 데이터 종류나 형태를 나타낸다.
- 클래스를 사용하면 int, String과 같은 타입을 직접 만들 수 있다.
- 사용자가 직접 정의하는 사용자 정의 타입을 만들려면 설계도가 필요하다. __이 설계도가 바로 클래스이다.__
- 설계도인 클래스를 사용해서 __실제 메모리에 만들어진 실체를 객체 또는 인스턴스라 한다.__
- 클래스를 통해서 사용자가 원하는 종류의 데이터 타입을 마음껏 정의할 수 있다.

#### 객체 생성
- new는 새로 생성한다는 뜻이다.
- 객체를 생성할 때는 new 클래스명()을 사용하면 된다. 마지막에 ()를 꼭 추가해줘야 한다.
- 멤버변수를 가지고 있다면 변수를 사용하는데 필요한 메모리 공간도 함께 확보한다.

#### 참조값 보관
- 객체를 생성하면 자바는 메모리 어딘가에 있는 이 객체에 접근할 수 있는 참조값(주소)을 반환한다.
- new 키워드를 통해 객체가 생성되고 나면 참조값을 반환한다.
- 선언한 변수에 생성된 객체의 참조값을 보관한다.
__참조값을 변수에 보관해야 하는 이유__
- 객체를 생성하는 new 클래스명() 코드 자체에는 아무런 이름이 없다.
- 클래스 기반으로 메모리에 실제 객체를 만드는 것이다. 따라서 생성한 객체에 접근할 수 있는 방법이 필요하다.
- 이런 이유로 객체를 생성할 때 반환되는 참조값을 어딘가에 보관해두어야 한다.
- 앞서 변수에 참조값을 저장해두었으므로 저장한 참조값을 통해서 실제 메모리에 존재하는 객체에 접근할 수 있다.

#### 클래스
- 클래스는 객체를 생성하기 위한 '틀' 또는 '설계도'이다.
- 객체가 가져야 할 속성(변수)과 기능(메서드)를 정의한다.

#### 객체
- 객체는 클래스에서 정의한 속성과 기능을 가진 실체이다.
- 객체는 서로 독립적인 상태를 가진다.

#### 인스턴스
- 인스턴스는 특정 클래스로부터 생성된 객체를 의미한다.
- 인스턴스는 주로 객체가 어떤 클래스에 속해 있는지 강조할 때 사용한다.
